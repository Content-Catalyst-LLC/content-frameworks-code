# Content Frameworks Code

Open computational companion repository for the Content Frameworks knowledge series.

This repository supports articles on content frameworks, pillar pages, topic clusters, narrative pathways, educational scaffolding, research communication, policy explanation, message architecture, audience insight, strategic analysis frameworks, editorial systems, and digital knowledge architecture.

## Repository Structure

- `articles/content-frameworks-structure-strategy-knowledge-systems/`
  - `python/` — pillar maps, internal-link graphs, metadata audits, and article-cluster examples
  - `r/` — framework-library comparison, content-audit summaries, and coverage analysis
  - `sql/` — schemas for pillars, articles, clusters, frameworks, templates, links, references, and editorial status
  - `docs/` — article notes and modeling principles
  - `data/` — synthetic datasets
  - `outputs/` — generated outputs
  - `notebooks/` — notebook placeholders

- `shared/`
  - `schemas/` — reusable content-system schemas
  - `templates/` — article, pillar, excerpt, metadata, and framework templates
  - `taxonomies/` — taxonomy examples and category structures
  - `editorial-models/` — editorial workflow models
  - `content-audits/` — reusable content-audit templates
  - `framework-libraries/` — reusable framework comparison tables
  - `governance/` — ethical and editorial governance notes
  - `tests/` — future validation tests

## Modeling Principles

1. Do not confuse a framework with understanding.
2. Distinguish templates, models, taxonomies, and strategic frameworks.
3. Make framework purpose explicit.
4. Match framework type to communication problem.
5. Preserve complexity where simplification would distort meaning.
6. Treat internal links as knowledge infrastructure.
7. Use metadata to support maintenance, not bureaucracy.
8. Use computation to make editorial systems inspectable and revisable.
