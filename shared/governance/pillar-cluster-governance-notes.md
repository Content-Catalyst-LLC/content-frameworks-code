# Pillar-Cluster Governance Notes

Pillar-cluster reviews should check:

- whether the pillar explains the topic rather than only listing links
- whether cluster articles have clear roles
- whether published cluster articles link back to the pillar
- whether the pillar links to published cluster articles
- whether metadata is complete
- whether planned articles still belong in the structure
- whether there are duplicate or overlapping articles
- whether internal links support reader pathways
- whether references and GitHub links are current
- whether the cluster is ready, developing, or early-stage
