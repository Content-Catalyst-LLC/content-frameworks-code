# Modeling Principles

1. Metadata fields should support decisions, workflows, reader needs, or governance requirements.
2. Article slugs should align with URLs and repository folder paths.
3. Footer navigation should be validated against the article-map order.
4. Image metadata should support accessibility, media-library management, and editorial consistency.
5. Reference metadata should support evidence review and source governance.
6. Status, article type, cluster, and severity values should use controlled vocabularies.
7. Review cycles should depend on article type and content risk.
8. Metadata completeness is a readiness signal, not a substitute for editorial quality.
9. Content systems should preserve relationships among articles, images, references, repositories, and governance records.
10. Governance queues should convert metadata failures into reviewable editorial actions.
