# Modeling Principles

1. A conceptual model is a selective representation, not the full communication situation.
2. Model quality depends on fit between model type and communication question.
3. Required elements should vary by model type.
4. Linear models are useful but should not be treated as universal.
5. Audience interpretation should be represented when meaning-making matters.
6. Feedback should be represented when communication evolves through response.
7. Evidence and limitations should be visible when models support public explanation.
8. Power and institutional context should be visible when they shape meaning.
9. Abstraction risk should be reviewed because clean models can hide complexity.
10. Model-readiness scores are governance signals, not proof of communication quality.
