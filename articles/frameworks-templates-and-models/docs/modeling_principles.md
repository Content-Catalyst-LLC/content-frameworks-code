# Modeling Principles

1. A framework organizes reasoning and interpretation.
2. A template standardizes repeatable output.
3. A model represents relationships, mechanisms, systems, or abstractions.
4. A method defines repeatable procedure.
5. A workflow coordinates people, tools, stages, and accountability.
6. Each structure type should be evaluated by the standard appropriate to its role.
7. Declared type should match observed use.
8. Misclassification should trigger governance review.
9. Templates should not replace strategy.
10. Models should not be mistaken for reality.
11. Frameworks should not be reduced to rigid fill-in forms.
12. AI-generated structures require classification, evidence review, and governance.
