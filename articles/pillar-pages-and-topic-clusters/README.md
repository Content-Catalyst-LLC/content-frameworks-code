# Pillar Pages and Topic Clusters

This companion directory supports the article **"Pillar Pages and Topic Clusters."**

The article explains how pillar pages, topic-cluster articles, internal links, metadata, and governance turn a collection of pages into a scalable knowledge architecture.

## Core Claim

Pillar pages and topic clusters are not only SEO devices. They are content-framework structures for organizing complex knowledge, supporting reader pathways, planning editorial coverage, auditing gaps, governing metadata, and scaling publications without losing coherence.

## Catalyst Canvas-Style Professional Workflow Layer

The workflows in this folder treat pillar-cluster architecture as an auditable content-system capability:

- pillar and cluster article inventory
- internal-link graph diagnostics
- pillar coverage density
- cluster readiness scoring
- metadata completeness checks
- orphan article detection
- weak-link and thin-cluster detection
- taxonomy coverage summaries
- article role validation
- governance review queue generation
- Catalyst Canvas-style catalog exports
- reproducible CSV, JSON, Markdown, SQL, Java, Python, and R outputs

## Repository Structure

- `config/` — pillar-cluster rules and governance thresholds
- `html/` — GitHub block, footer navigation, method-wrapper snippets, and reusable article components
- `css/` — reusable styling patterns
- `php/` — WordPress-oriented helper snippets
- `java/` — pillar-cluster model and validation logic
- `python/` — Catalyst Canvas-style pillar-cluster audit engine
- `r/` — cluster coverage, link density, and metadata readiness analysis
- `sql/` — schema and analytical queries
- `docs/` — article notes, modeling principles, governance documentation, and product notes
- `data/` — synthetic datasets
- `outputs/` — generated outputs
- `notebooks/` — notebook placeholders

## Quick Start

From this article directory:

```bash
python3 python/run_all_pillar_cluster_workflows.py
Rscript r/run_all_pillar_cluster_workflows.R
javac java/PillarClusterModel.java && java -cp java PillarClusterModel
sqlite3 outputs/pillar_cluster.sqlite < sql/pillar_cluster_schema.sql
sqlite3 -header -csv outputs/pillar_cluster.sqlite < sql/pillar_cluster_queries.sql > outputs/tables/sql_pillar_cluster_report.csv
```
